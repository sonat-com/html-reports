What's new in Sonat

May 2026 — new features, improvements & fixes

Here's everything we shipped in May — from a new link-health dashboard and fullscreen image viewing to root-domain support and faster page loads. Every update is built to help you create better documentation and give your readers a smoother experience.

✨ New Features

🔗

Reference Links Manager

A new dashboard in the editor shows every outbound link in your documentation, flags broken ones, and lets you mark a link as intentionally accepted so it stops showing as an issue. A live broken-link count in the toolbar means you always know the health of your content at a glance.

🖼️

Fullscreen Image Viewer

Readers can now click any image in your published documentation to open it fullscreen — perfect for detailed diagrams, screenshots, and illustrations where the inline view isn't enough.

🌐

Root-Domain (Apex) Support for Custom Domains

You can now point your root domain — for example company.com — directly at your documentation. Previously only subdomains such as docs.company.com were supported.

📄

PDF Export for Standalone Pages

Standalone pages can now be downloaded as PDFs — the same Download PDF button your readers already use on full documents is now available on individual pages too.

🤖

AI-Powered Search Answers

Search in your published documentation now shows an AI-generated direct answer at the top of the results, so readers get what they need immediately instead of scanning through a list.

💬

Assistant Widget: Open on Any Page

The embeddable Sonat Assistant widget can now be set to open on any standalone page or topic by default — ideal for help widgets that should go straight to a specific FAQ or getting-started guide.

🗂️

Custom Navigation Menus in Published Docs

The navigation menus you build in the Menu Builder now render fully in your published documentation — including nested items and custom links, exactly as you designed them.

🚀 Improvements

📂

Google Docs import respects your current folder

When you create or import a Google Doc, it now lands in the folder you're browsing — not at the root.
⚡

Faster public documentation pages

A round of performance work made public viewer pages noticeably faster to load — benefiting every reader of your documentation.
👁️

Clear preview-mode indicator

When previewing unpublished content, a visible banner now makes it obvious you're in preview mode — no more mixing it up with the live version.
🖼️

Images render exactly as authored

Image sizing, alignment, and spacing in published docs now match exactly what you set in the editor.
📱

Smoother navigation on mobile

The topic menu on phones now lives in the navigation bar, making it easier for readers to move around your documentation on smaller screens.
💡

Sidebar tooltips for long topic names

Long topic titles in the sidebar now show the full name on hover, even when the text is truncated.
📑

PDF downloads use language-specific titles

For translated documents, downloaded PDFs are now named after the correct translated title for each language.
🌙

Better dark mode in the viewer

Link colors in the sidebar are now tuned for dark mode, keeping navigation clear and readable for readers who prefer a dark theme.

🛠️ Fixes & Polish

  • Trying to delete a document or page that's set as the default now shows a clear, helpful message.
  • Only published content can be set as the default page, preventing broken default links.
  • Switching between document versions now updates the address bar correctly, so shared links open the right version.
  • Copying a document now copies all of its images too, so the duplicate is fully self-contained.
  • Version navigation links now resolve correctly for multi-version documents on custom domains.
