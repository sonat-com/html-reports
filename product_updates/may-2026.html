Here's a summary of everything we shipped in May — from powerful new link management tools to fullscreen image viewing, apex domain support, and faster load times. Every update is built to help you create better documentation and give your readers a better experience.

✨ New Features

🔗

Reference Links Manager

A new dashboard in the editor shows every outbound link in your documentation, flags broken ones, and lets you mark a link as intentionally accepted so it stops appearing as an issue. A broken-link count appears right in the toolbar so you always know the health of your content at a glance.

🖼

Fullscreen Image Viewer

Readers can now click any image in your published documentation to view it fullscreen. Perfect for detailed diagrams, screenshots, and technical illustrations where the small inline view isn't enough.

🌐

Root Domain (Apex) Custom Domain Support

You can now point your root domain — e.g. company.com — directly to your Docer documentation. Previously only subdomains like docs.company.com were supported.

📄

PDF Export for Standalone Pages

Standalone "Page" topics can now be downloaded as PDFs — the same Download PDF button your readers already use on full documents is now available on individual pages too.

🤖

AI-Powered Search Results

Search results in your published documentation now include an AI-generated direct answer at the top, so readers get the answer immediately instead of having to scan multiple results.

💬

Web Assistant: Open on Any Topic

The embeddable Sonat Assistant widget can now be configured to open on any standalone page or topic by default — not just full documents. Ideal for help widgets that should go straight to a specific FAQ or getting-started page.

🗂

Custom Navigation Menus in Published Docs

The custom navigation menus you build in the Menu Builder are now fully rendered in your published documentation viewer, including nested items and custom links — exactly as you designed them.

🚀 Improvements

📂

Google Docs import respects your current folder

When you create or import a Google Doc, it now lands in the folder you're browsing — not at the root.
⚡

Faster public documentation pages

A Lighthouse performance audit resulted in measurably faster load times on all public viewer pages — benefiting every reader of your documentation.
👁

Clear preview mode indicator

When previewing unpublished content, a visible banner now makes it immediately clear you're in preview mode — no more confusion with the live version.
🖼

Images render exactly as authored

Image sizing, alignment, and spacing now match exactly what you set in the editor — ensuring your published documentation looks exactly as intended.
💡

Sidebar tooltips for long topic names

Long topic titles in the sidebar now show a tooltip on hover so readers can always read the full title, even when it's truncated.
📑

PDF downloads use language-specific titles

When a document has multiple translations, downloaded PDFs are now named after the correct translated title for each language — no more generic English filenames for translated content.
🌙

Better dark mode in the viewer

Link colors in the sidebar are now properly adapted for dark mode, making navigation readable and accessible for readers who prefer a dark theme.

🐛 Bug Fixes

  • Deleting a document or topic that's set as a default now shows a clear, helpful error message instead of failing silently.
  • Unpublished content can no longer be accidentally set as the default page for a folder or account.
  • Switching between document versions now updates the URL correctly so sharing the link goes to the right version.
  • Copying a document now also copies all of its images, so the duplicated doc is fully self-contained.
  • Version navigation links in the sidebar now resolve to the correct URL for multi-version documents on custom domains.
