Here's everything we shipped over the past month — from a brand-new API access layer to smarter broken-link detection and a more polished viewer experience. Every change is designed to help your team create better documentation and give your readers a smoother experience.

✨  New Features

🔑

API Tokens

You can now generate role-bearing API tokens directly from your account settings. Use them to integrate Docer with your own tools, automate publishing workflows, or connect third-party apps — without sharing your personal credentials.

🔗

Broken Link Manager

A new dashboard in the editor surfaces every outbound link in your documentation and flags broken ones. A live broken-link count is shown right in the toolbar so you always know the health of your content. You can also mark specific links as intentionally accepted to keep the dashboard tidy.

🖼

Fullscreen Image Preview

Readers can now click any image in your published documentation to view it fullscreen. Ideal for detailed diagrams, screenshots, and technical illustrations where the inline view isn't enough.

📄

PDF Export for Standalone Pages

Standalone pages — topics not backed by a Google Doc — can now be downloaded as PDFs. The same Download PDF button your readers already know is now available on every page type.

🤖

AI Answer in Search Results

Search results in published documentation now surface an AI-generated direct answer at the top, so readers get the information they need immediately — without scanning through multiple results.

📋

Template Library

A curated template library is now live. Browse ready-to-use documentation templates to jumpstart any new manual, knowledge base, or help centre — no blank page anxiety.

🚀  Improvements

📂

Google Docs import respects your current folder

When you create or import a Google Doc, it now lands in the folder you're browsing — not at the root level.
⚡

Faster public documentation pages

A full Lighthouse performance audit resulted in measurably faster load times on all public viewer pages — every reader benefits.
🔍

Improved search bar in the viewer

The search bar is now always visible in the navigation and properly highlights matching text in results — making it easier for readers to find exactly what they're looking for.
👁

Clear preview mode indicator

A visible banner now appears when you're previewing unpublished content — no more confusion between draft previews and the live version your readers see.
🖼

Images render exactly as authored

Image sizing, alignment, and spacing now faithfully match what you set in the editor — published documentation looks exactly as intended.
💡

Sidebar tooltips for long topic names

Long topic titles in the sidebar now show a tooltip on hover so readers can always see the full name, even when it's truncated.

🐛  Bug Fixes

·

Safer content deletion — Deleting a document or topic that is set as the default landing page is now blocked with a clear, friendly error message to prevent accidental breakage.
·

Google Docs import spinner — The import loading indicator is now centred and displays as a proper fullscreen overlay, so it's never missed.
·

PDF cover page titles — PDF exports now use the correct document title in each language on the cover page and in the filename.
·

Dark mode link colours — Link colours in the sidebar now render correctly in dark mode.
·

SEO: accurate 404 pages — Documentation pages that don't exist now return a proper 404 response to search engines, keeping your SEO health clean and preventing outdated links from being indexed.
·

More reliable PDF exports — PDF generation is now significantly more robust: transient failures auto-retry, and exports that would previously time out silently now complete reliably.
